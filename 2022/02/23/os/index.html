Operating system

In this passage I will mainly record my learning process of OS, including the meaning of certain terms, the big idea of what I can do with this knowledge and answer questions that may be asked in later interviews.

I’ve heard Remzi, professor in UWM, has rewritten a great textbook: . (https://pages.cs.wisc.edu/~remzi/OSTEP/) and courses link in (https://www.bilibili.com/video/BV19J411t7te?spm_id_from=333.999.0.0). These materials are gonna be my preliminary learning source.

Chapter 1 Introduction

Lecture 1: Introduction to the class, including the basics of what an OS does, and the major themes of the course: virtualization, concurrency, and persistence. Overview of how the class works. Finally, introduction to CPU virtualization, and start of discussion on mechanisms to support it.

Chapter 2

Lecture 2: Recapitulation of mechanisms to support CPU virtualization, and completion of said topic. Introduction to CPU scheduling (policies). Introduction to homeworks for CPU virtualization chapters.

Chapter

Lecture 3: CPU scheduling policy (continued). Focus on multi-level feedback queue (Part 1). Introduction to virtual memory, including simplest mechanism called dynamic relocation (Part 2). Wrap up dynamic relocation, talk about segmentations (Part 3).

Chapter

Lecture 4: Review of dynamic relocation and segmentation, and problems. Intro to paging (Part 1). More on paging mechanisms, and intrinsic problems (Part 2). Introduction to TLBs (Part 3).

Chapter

Lecture 5: Review of paging and its problems. How to handle the fact that linear page tables are too big? Hint: multi-level page tables (Part 1). How to handle the fact that programs might use more memory than is physically available? Hint: swap to disk. (Part 2).

Chapter

Lecture 6: Intro to concurrency (Part 1). Understanding a simple data race, the lack of mutual exclusion, and indeterminate outcomes (Part 2). How to build a simple spin lock using atomic exchange, how to add locks to a list or hash table (Part 3).

Chapter

Lecture 7: More concurrency. Review of spin locks and their problems. Ticket locks, yield(), and park()/unpark() (Part 1). Wrap up of locks (for now). Introduction to condition variables, and the fork/join problem (Part 2). Continued fork/join, intro to bounder buffer (producer/consumer) problem (Part 3).

Chapter

Lecture 8: Remembering producer/consumer (Part 1). Ending producer/consumer, intro to semaphores (Part 2).

Chapter

Lecture 9: Bugs and deadlock (Part 1). Review (Part 2).

Chapter

Lecture 10: Introduction to Persistence. The Hard Disk Drive. Disk Scheduling.

Chapter

Lecture 11: RAID. Levels 0, 1, 4, and 5. File System API and Abstractions.

Chapter

Lecture 12: Intro to on-disk file system structures (Part 1). Access methods (Part 2). Caching + Stuff (Part 3).

Chapter

Lecture 13: Intro to crash consistency (Part 1). Crash consistency and journaling (Part 2). Fast File System (Part 3).

Chapter

Lecture 14: LFS intro (Part 1). More LFS, including garbage collection (Part 2). Flash-based SSDs (Part 3). So-called words of wisdom (Part 4).
